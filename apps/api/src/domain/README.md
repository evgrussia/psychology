# Domain Layer

Этот слой содержит бизнес-логику и доменные правила согласно принципам Clean Architecture и DDD.

## Структура

- **Entities** - доменные сущности с уникальной идентичностью
- **Value Objects** - неизменяемые объекты-значения
- **Aggregates** - агрегаты с корневыми сущностями (Aggregate Roots)
- **Domain Services** - сервисы для бизнес-логики, не принадлежащей конкретной сущности
- **Repository Interfaces** - интерфейсы для доступа к данным (реализации в infrastructure)
- **Domain Events** - события домена (`events/`)

## Принципы

- ✅ Не зависит от внешних фреймворков (кроме стандартных библиотек языка)
- ✅ Содержит только бизнес-логику
- ✅ Интерфейсы репозиториев определены здесь, реализации - в infrastructure
- ✅ События домена публикуются через EventBus
