# /tone-check

## Назначение
Проверить текст на соответствие бережному тону и редакционной политике проекта

## Формат
```
/tone-check
<текст для проверки>
```

## Что проверяется

### Запрещённые формулировки
- Медицинские термины (лечим, диагноз, симптомы)
- Обещания и гарантии (гарантируем результат, всё будет хорошо)
- Давление и директивность (вы должны, срочно, не откладывайте)
- Обесценивание (не так страшно, пустяки)
- Marketing-агрессия (не упустите, только сегодня)

### Позитивные критерии
- Эмпатичный тон
- Уважение к автономии пользователя
- CTA с объяснением пользы
- Дисклеймеры где нужно

## Пример использования

```
/tone-check
Запишитесь на консультацию прямо сейчас! Вам срочно нужна помощь специалиста!
```

### Ожидаемый результат:
```yaml
issues:
  - type: pressure
    found: "прямо сейчас"
    suggestion: "когда будете готовы"
  - type: pressure
    found: "срочно нужна"
    suggestion: "может быть полезно"
  - type: tone
    found: "!"
    suggestion: "Проверьте: нужен ли восклицательный знак?"

suggested_rewrite: |
  Записаться на консультацию
  Если чувствуете готовность — консультация может помочь разобраться в ситуации.
```
