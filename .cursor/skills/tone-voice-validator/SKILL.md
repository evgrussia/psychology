# Tone & Voice Validator

## Назначение
Проверка текстового контента на соответствие редакционной политике и бережному тону проекта «Эмоциональный баланс». Выявление запрещённых формулировок и генерация корректных альтернатив.

## Когда применять
- При написании UI copy
- При создании контента (статьи, ресурсы, результаты тестов)
- При написании email-шаблонов
- При формулировке CTA
- При написании сообщений об ошибках
- В code review контентных изменений

## Принципы бережного тона

### Главный принцип
**Эмпатично, без давления, не директивно**

Пользователь — в уязвимом состоянии. Тексты должны:
- Создавать безопасное пространство
- Не оказывать давления
- Уважать автономию
- Нормализовать переживания
- Предлагать, а не требовать

## Алгоритм проверки текста

### Шаг 1: Сканирование на запрещённые слова

```python
# Псевдокод проверки
def check_text(text: str) -> ValidationResult:
    issues = []
    
    # Проверить медицинские термины
    for term in MEDICAL_TERMS:
        if term in text.lower():
            issues.append(Issue(
                type="medical_term",
                found=term,
                suggestion=ALTERNATIVES[term]
            ))
    
    # Проверить обещания
    for phrase in PROMISE_PHRASES:
        if phrase in text.lower():
            issues.append(Issue(
                type="promise",
                found=phrase,
                suggestion=ALTERNATIVES[phrase]
            ))
    
    # Проверить давление
    for phrase in PRESSURE_PHRASES:
        if phrase in text.lower():
            issues.append(Issue(
                type="pressure",
                found=phrase,
                suggestion=ALTERNATIVES[phrase]
            ))
    
    return ValidationResult(issues=issues)
```

### Шаг 2: Контекстуальная проверка

- CTA без объяснения пользы → добавить объяснение
- Результат теста без дисклеймера → добавить дисклеймер
- Сообщение об ошибке с обвинением → переформулировать

### Шаг 3: Проверка дисклеймеров

| Контекст | Требуемый дисклеймер |
|----------|---------------------|
| Квиз/тест | "Это не диагноз, а ориентир" |
| Статья о здоровье | "Материал носит информационный характер" |
| Результат высокого уровня | Блок экстренной помощи |

## Чек-лист проверки

### Для UI Copy

```markdown
## Tone Validation: [Component/Screen Name]

### Запрещённые формулировки
- [ ] Нет медицинских терминов (лечим, диагноз, симптомы)
- [ ] Нет обещаний результата (гарантируем, точно поможет)
- [ ] Нет давления (должны, срочно, не откладывайте)
- [ ] Нет обесценивания (не так страшно, пустяки)
- [ ] Нет агрессивного маркетинга (не упустите, только сегодня)

### Позитивные критерии
- [ ] Тон эмпатичный
- [ ] Уважение к автономии пользователя
- [ ] CTA с объяснением пользы
- [ ] Сообщения об ошибках без обвинения
- [ ] Дисклеймеры где нужно
```

### Для статей/контента

```markdown
## Content Tone Validation: [Article Title]

### Обязательные элементы
- [ ] Дисклеймер "не заменяет консультацию"
- [ ] Нет директивных советов ("вы должны")
- [ ] Использован смягчающий язык ("может помочь", "некоторым подходит")
- [ ] Нет обещаний результата

### Тон
- [ ] Информирует, а не предписывает
- [ ] Нормализует переживания
- [ ] Предлагает варианты, а не единственное решение
```

### Для email-шаблонов

```markdown
## Email Tone Validation: [Template Name]

- [ ] Subject line без давления
- [ ] Приветствие тёплое, не формальное
- [ ] CTA без агрессии
- [ ] Возможность отписаться очевидна
- [ ] Нет манипулятивных приёмов
```

## Примеры перефразирования

### CTA (Call to Action)

```markdown
❌ "Запишитесь на консультацию прямо сейчас!"
   Проблема: давление ("прямо сейчас"), восклицательный знак

✅ "Записаться на консультацию"
   Или: "Если чувствуете готовность — можно записаться на консультацию"
```

```markdown
❌ "Не откладывайте решение своих проблем!"
   Проблема: давление, обвинение ("своих проблем")

✅ "Консультация может помочь разобраться в ситуации"
```

### Результаты тестов

```markdown
❌ "У вас высокий уровень тревоги. Вам срочно нужна помощь специалиста!"
   Проблема: директивность, давление

✅ "Результат показывает высокий уровень тревоги. 
    Это ориентир, не диагноз. 
    Если результаты вызывают беспокойство — стоит обсудить их со специалистом."
```

```markdown
❌ "Тест показал, что вы больны депрессией."
   Проблема: медицинский термин, диагноз

✅ "Результат указывает на признаки, которые могут быть связаны с подавленным настроением.
    Для более точного понимания рекомендуется консультация со специалистом."
```

### Сообщения об ошибках

```markdown
❌ "Ошибка! Вы ввели неправильный email!"
   Проблема: обвинение пользователя, агрессивный тон

✅ "Кажется, email введён некорректно. Попробуем ещё раз?"
```

```markdown
❌ "Этот слот уже занят. Выберите другой."
   Проблема: сухой тон, нет эмпатии

✅ "К сожалению, это время уже занято. Давайте подберём другой слот?"
```

### Empty States

```markdown
❌ "У вас нет записей в дневнике. Начните прямо сейчас!"
   Проблема: давление

✅ "Здесь будут ваши записи в дневнике. 
    Начните, когда будете готовы — это займёт пару минут."
```

### Подтверждения

```markdown
❌ "Поздравляем! Вы записались на консультацию!"
   Проблема: слишком "праздничный" тон для чувствительной темы

✅ "Консультация подтверждена. До встречи!"
```

## Специальные контексты

### Кризисный контекст

При высоком уровне тревоги/кризиса:

```markdown
✅ Правильно:
"Если вам сейчас очень тяжело — вы не одни.
Телефон доверия: 8-800-2000-122 (бесплатно, круглосуточно)

Запись на консультацию может подождать. 
Позвоните на линию поддержки — там помогут прямо сейчас."

❌ Неправильно:
"Вам срочно нужна помощь! Запишитесь на консультацию немедленно!"
```

### Отказ от услуги

```markdown
✅ Правильно:
"Если сейчас не время — это нормально. 
Вы всегда можете вернуться, когда будете готовы."

❌ Неправильно:
"Жаль, что вы уходите. Не упустите шанс!"
```

### Напоминания

```markdown
✅ Правильно:
"Напоминаю о консультации завтра в 15:00.
Если нужно перенести — напишите, подберём другое время."

❌ Неправильно:
"Не забудьте о консультации! Не пропустите!"
```

## Автоматизация проверки

### Regex паттерны для линтера

```python
# Паттерны для автоматической проверки
FORBIDDEN_PATTERNS = [
    # Медицинские термины
    (r'\bлечи[мтьшь]?\b', "medical", "Замените на 'работаем с' или 'помогаем'"),
    (r'\bдиагноз\b', "medical", "Замените на 'ориентир' или 'результат'"),
    (r'\bсимптом[ыа]?\b', "medical", "Замените на 'признаки' или 'проявления'"),
    
    # Давление
    (r'\bвы должны\b', "pressure", "Замените на 'вы можете' или 'один из вариантов'"),
    (r'\bсрочно\b', "pressure", "Уберите или замените на 'когда будете готовы'"),
    (r'\bобязательно\b', "pressure", "Замените на 'если хотите' или уберите"),
    (r'\bне откладывайте\b', "pressure", "Уберите или смягчите"),
    
    # Обещания
    (r'\bгарантируем\b', "promise", "Замените на 'часто помогает'"),
    (r'\bточно поможет\b', "promise", "Замените на 'может быть полезно'"),
    (r'\bвсё будет хорошо\b', "promise", "Уберите — пустое обещание"),
    
    # Marketing агрессия
    (r'\bне упустите\b', "marketing", "Замените на нейтральный CTA"),
    (r'\bтолько сегодня\b', "marketing", "Уберите — манипуляция"),
    (r'!\s*$', "tone", "Проверьте: нужен ли восклицательный знак?"),
]
```
