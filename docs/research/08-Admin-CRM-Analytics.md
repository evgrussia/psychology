# 08 — Админ‑панель, CRM‑лиды и аналитика

## 1) Зачем админка
Админка нужна не “для красоты”, а чтобы:
- управлять расписанием и услугами без разработчиков,
- видеть путь пользователя (квизы → TG → запись),
- быстрее готовить контент и ответы,
- соблюдать конфиденциальность (модерация, права доступа).

## 2) Роли и доступы (минимально)
- **Психолог (owner)**: полный доступ
- **Ассистент** (опционально): расписание/лиды/уведомления, без доступа к чувствительным заметкам
- **Контент‑редактор**: статьи/ресурсы/лендинги, без доступа к клиентским данным

## 3) Модуль расписания
### 3.1 Функции
- создание слотов (правила повторения)
- исключения (отпуск/перерывы)
- разные длительности по услугам
- буферы между сессиями
- импорт/синхронизация календаря (если есть интеграция)

### 3.2 Представления
- календарь день/неделя
- список ближайших встреч

## 4) Модуль услуг
- CRUD услуг
- цены/депозиты
- пакеты (кол-во встреч, срок действия)
- политика переносов/отмен (на уровне услуги или глобально)

## 5) CRM‑лиды (воронка)
### 5.1 Источники лидов
- прохождение квиза/теста (анонимно)
- подписка Telegram
- заявка “нет слотов → лист ожидания”
- “анонимный вопрос”
- запись (booking)

### 5.2 Пайплайн (статусы)
- new (новый контакт)
- engaged (прошёл интерактив / подписался)
- booking_started
- booked_confirmed
- paid
- completed_session
- follow_up_needed
- inactive

### 5.3 Карточка лида (поля)
- источник/UTM
- сегмент (тревога/выгорание/…)
- события (timeline)
- заметка ассистента (не терапевтическая)
- согласия (галочки)

## 6) Контент‑менеджмент (CMS‑часть)
### 6.1 Типы контента
- статьи
- микро‑заметки
- ресурсы (упражнения/аудио/чек‑листы)
- лендинги под запросы
- страницы "как проходит", "обо мне"
- **подборки контента** (`/curated/`) — NEW
- **словарь терминов** (`/glossary/`) — NEW

### 6.2 Редактор и QA‑чеклист
Чеклист перед публикацией:
- есть дисклеймер
- тон не директивный
- есть "первый шаг" (упражнение/ресурс)
- есть CTA (Telegram/запись) с объяснением
- внутренняя перелинковка (2–5 ссылок)

### 6.3 Медиа-библиотека — NEW
- загрузка изображений/аудио/PDF
- организация по папкам/тегам
- превью и метаданные
- использование в контенте через выбор из библиотеки

## 6A) Управление подборками контента — NEW
### 6A.1 Зачем
- Навигация по большому каталогу статей/ресурсов
- Сценарное мышление ("что почитать про отношения")
- Удержание через "что ещё полезного"

### 6A.2 Функции админки
- Создание/редактирование подборок
- Добавление элементов (статьи/ресурсы/упражнения/квизы)
- Настройка порядка (drag & drop)
- Категоризация (по теме/формату/цели/контексту)
- Превью подборки

## 6B) Управление словарём терминов — NEW
### 6B.1 Зачем
- SEO (термины как точки входа)
- Доверие (экспертность и прозрачность подхода)
- Образование пользователей

### 6B.2 Функции админки
- Создание/редактирование терминов
- Категоризация (подходы/методы | состояния | понятия/техники)
- Простой редактор с markdown
- Перелинковка на статьи/ресурсы (автоподсказки)
- Синонимы (для поиска)

## 7) Интерактивы: управление вопросами и результатами
### 7.1 Что должно настраиваться без кода (по возможности)
- тексты вопросов
- шкалы/варианты
- уровни результата и тексты рекомендаций
- CTA‑варианты
- **пороги уровней (low/moderate/high)** — NEW
- **варианты скриптов границ (BND-01)** — NEW
- **библиотека мини-ритуалов (добавление/удаление)** — NEW
- **настройки навигатора состояния (шаги/профили)** — NEW

### 7.2 Безопасность
- логирование событий без сохранения чувствительных текстов
- кризисные триггеры → обязательный сценарий

### 7.3 Превью интерактива — NEW
- Открытие интерактива "как видит пользователь" перед публикацией
- Проверка всех путей/результатов

## 8) Шаблоны сообщений (email/TG)
Типы шаблонов:
- подтверждение записи
- напоминания
- “как подготовиться”
- материалы после сессии
- “нет слотов” → лист ожидания

Требования:
- переменные (имя, время, ссылка)
- версия и история изменений

## 9) Модерация отзывов и историй
### 9.1 Поток
- получено → проверка согласия → анонимизация → опубликовано
- возможность снять с публикации

### 9.2 Поля согласия
- разрешение на публикацию
- уровень анонимности
- возможность отозвать согласие

### 9.3 Дашборд модерации UGC — NEW
**Метрики (из `docs/UGC-Moderation-Rules.md` раздел 8):**
- Время модерации (цель: <24 часа обычные, <4 часа кризис)
- Доля отклонённых (цель: <10%)
- Доля кризисных (следить за динамикой)
- Жалобы на модерацию (цель: 0)

**Очередь модерации:**
- Список pending/flagged вопросов
- Фильтры по статусу/триггерам
- История действий модератора

**События для аналитики:**
- `ugc_moderated` (status, duration_ms, moderator_role)
- `moderation_escalated` (reason)
- `ugc_answered` (time_to_answer_hours)

## 10) Аналитика и дашборды
### 10.1 Главные KPI
Из брифа:
- visit → booking
- visit → TG subscribe
- engage → lead
- возвраты (дневники/челлендж)
- **no-show rate** — NEW (требует события `appointment_outcome_recorded`)
- **retention D1/D7/D30** (Telegram и продукт) — NEW

### 10.2 Дашборды (минимум)
- **Воронка**: посещения → интерактив → TG → запись → оплата
- **Каналы**: SEO/Telegram/реферальный
- **Темы**: тревога/выгорание/…
- **Интерактивы**: completion rate, CTA rate
- **Слоты**: загрузка расписания, no‑show
- **Модерация UGC**: время модерации, доля отклонённых/кризисных — NEW
- **Контент**: популярность статей/ресурсов, CTA rate — NEW
- **Мероприятия**: регистрации, явка — NEW
- **Финансы**: GMV, AOV, динамика — NEW

### 10.3 События (event catalog)
См. `01-IA-and-UserJourneys.md` (раздел событий), плюс:
- `review_submitted` / `review_published`
- `waitlist_submitted`
- `template_sent`
- **`appointment_outcome_recorded`** (attended/no_show/canceled) — NEW
- **`tg_interaction`** (для retention Telegram) — NEW
- **`ugc_moderated`** / `moderation_escalated` / `ugc_answered` — NEW

### 10.4 Критичные события для метрик — NEW
**No-show rate:**
- Требуется обязательная реализация `appointment_outcome_recorded`
- Экран в админке для отметки исхода встречи после завершения
- Дашборд с фильтрами (по услуге/периоду)

**Retention Telegram (D1/D7/D30):**
- Требуется обязательная реализация `tg_interaction`
- Отправка при любом взаимодействии в боте (кнопка/команда/сообщение)
- Cohort retention table в аналитике

## 11) Аудит‑лог и безопасность
Минимум:
- кто и когда изменил цены/политику
- кто экспортировал данные
- кто удалил запись/дневник (если доступно)
