# Roadmap: P1-активности для релиза 2 — «Эмоциональный баланс»

**Версия:** v1.0  
**Дата:** 2026-01-07  
**Источник:** `docs/research/13-Competitive-Research-Psychology-Portals.md` (раздел 8), `docs/PRD.md` (раздел 6.3 backlog)

---

## Зачем этот документ

Фиксирует **план внедрения P1-активностей** из конкурентного исследования после релиза 1. Эти активности:
- повышают **удержание** (D1/D7/D30)
- создают **уникальность** ("небанальные" интерактивы)
- поддерживают **бережный тон** без давления

**Приоритизация:** после 30–60 дней работы релиза 1 оцениваем метрики и решаем, какие активности внедрять первыми.

## Детали активностей

### P1-01 — Письмо себе через 7 дней

#### Суть
Пользователь выбирает шаблон письма (поддержка/напоминание/цель) и получает его через 7 дней (Telegram/email).

#### Лайт-MVP (безопасный)
- **Только шаблоны** (5–7 вариантов), **без свободного текста**
- Примеры:
  - "Я хочу напомнить себе: ты справляешься"
  - "Я хочу напомнить себе: [выбор цели: отдохнуть/попросить помощь/обозначить границу]"
  - "Я хочу поддержать себя: ты делаешь, что можешь"
- Выбор → подтверждение → напоминание через 7 дней "перечитать/открыть"

#### Что НЕ делаем (риски)
❌ Свободный текст (риск чувствительных данных)  
❌ Хранение на сервере без явного согласия  
❌ Публикация писем  

#### Хранение
- **Вариант A (рекомендуемый):** только `template_id` + `scheduled_at`, текст шаблона подставляется при доставке
- **Вариант B:** если пользователь хочет персонализацию — локально (localStorage) или с явным согласием

#### Доставка
- Telegram (primary) или email (fallback)
- Возможность отменить напоминание

#### События
- `letter_scheduled` (template_id, delivery_channel, scheduled_date)
- `letter_delivered` (template_id)
- `letter_cancelled`

#### Ценность
- Возвращаемость: пользователь получает "теплое" напоминание через 7 дней
- Эмоциональная связь с продуктом

#### Критерии запуска
- Retention D7 <30% (нужны дополнительные механики возврата)
- Telegram-подписка активна и стабильна (доставка работает)

### P1-02 — Карта поддержки близкому

#### Суть
Сценарий для партнёра/родителя/друга: "что сказать/чего не говорить/как предложить помощь" + экстренная помощь.

#### Лайт-MVP
- **Короткий гайд** (1 страница):
  - Раздел "Что говорить" (5–7 фраз)
  - Раздел "Чего не говорить" (5–7 анти-примеров)
  - Раздел "Как предложить помощь" (3–5 шагов)
  - **Обязательный блок "Urgent support"** (когда нужна срочная помощь)
- Чек-лист "Я могу" (что может сделать близкий прямо сейчас)
- Ссылки на ресурсы (экстренная помощь, горячие линии)

#### Что НЕ делаем (риски)
❌ Инструкции "лечения"  
❌ Советы для ситуаций насилия (только ресурсы и безопасность)  
❌ Обещания результата  

#### Разделы
- **Карта для разных состояний:**
  - Тревога/паника
  - Выгорание
  - Депрессия (осторожно, только общие рекомендации)
  - Кризис (обязательно с экстренной помощью)

#### Доставка
- Страница на сайте (`/resources/support-map-for-loved-ones/` или `/curated/support-map/`)
- Экспорт PDF
- Сохранение в Telegram

#### События
- `support_map_viewed` (topic)
- `support_map_exported` (export_target)

#### Ценность
- Расширение аудитории (близкие становятся "амбассадорами")
- Доверие ("здесь заботятся и о близких")
- Снижение стигмы

#### Критерии запуска
- Каталог контента >10 статей (чтобы была база для перелинковки)
- Есть ресурсы "urgent support" (обязательно)

### P1-03 — Кризисная карточка (план безопасности)

#### Суть
Пользователь собирает карточку из готовых блоков: "что помогает", "кому позвонить", "что сделать прямо сейчас", "сигналы опасности".

#### Лайт-MVP (без персональных данных)
- **Универсальная карточка** (не требует ввода контактов):
  - "Что помогает мне" (выбор из списка: дыхание/прогулка/разговор/музыка/другое)
  - "Кому позвонить" (универсальный список: 112, телефон доверия, близкий человек)
  - "Что сделать прямо сейчас" (выбор из 3–5 шагов)
  - "Сигналы опасности" (когда обратиться за помощью)
- Экспорт PDF или сохранение в Telegram (без сервера)

#### Что НЕ делаем (риски)
❌ Хранение персональных контактов/адресов на сервере (без явного согласия)  
❌ Публикация карточек  
❌ Советы "лечения"  

#### Юридическая/этическая вычитка
- Обязательна перед запуском
- Дисклеймер: "не заменяет профессиональную помощь"
- Блок "urgent support" должен быть первым

#### Доставка
- Страница на сайте (`/start/crisis-card/` или `/resources/safety-plan/`)
- Экспорт PDF (рекомендуемый)
- Сохранение в Telegram (опционально)

#### События
- `crisis_card_start`
- `crisis_card_completed`
- `crisis_card_exported` (export_target)

#### Ценность
- Повышает ощущение безопасности
- Соответствует best practices (safety planning)
- Может быть использована на консультации

#### Критерии запуска
- Юридическая проверка пройдена
- Кризисный режим протестирован и стабильно работает
- Есть ресурсы "urgent support"

### P1-04 — Тихая геймификация без стриков

#### Суть
Вместо "серий" и давления — "мягкий прогресс": отметка "я сделал(а) один маленький шаг" + подбор следующего шага.

#### Лайт-MVP
- **3 статуса состояния** (пользователь выбирает каждый день или по желанию):
  - "Сейчас тяжело" → рекомендации: отдых/минимальные шаги
  - "Нормально" → рекомендации: поддержание/практики
  - "Есть ресурс" → рекомендации: новые практики/углубление
- **"Один маленький шаг":** пользователь отмечает, что сделал (без обязательства делать каждый день)
- **Подбор следующего шага** (на основе статуса и истории)
- **Без стриков/серий** (только "сколько шагов сделано всего", без давления)

#### Что НЕ делаем (риски)
❌ Стрики/серии (усиливают тревогу при "срыве")  
❌ Сравнение с другими  
❌ "Ты молодец/ты справляешься" (может быть триггером)  

#### Механика
- Пользователь заходит в ЛК или Telegram
- Видит вопрос: "Как сегодня?"
- Выбирает статус → получает рекомендацию "один шаг"
- Отмечает "сделал(а)" (опционально)
- Видит "всего шагов: N" (без сравнения/давления)

#### Напоминания
- Частота настраивается пользователем (1 раз в день / 2–3 раза в неделю / только по запросу)
- **Всегда легко отключить**

#### События
- `status_check_in` (status=difficult|ok|resourceful)
- `small_step_suggested` (step_type)
- `small_step_completed` (step_type)
- `gamification_disabled` (reason)

#### Ценность
- Удержание без стыда
- Ближе к тону проекта ("бережно, без давления")
- Снижает барьер "я не справляюсь"

#### Критерии запуска
- Дневники работают стабильно (база для "шагов")
- Retention D7 <30% (нужны мягкие механики возврата)
- Напоминания (Telegram/email) настроены и работают

## Приоритизация: когда запускать

### Критерии решения (после 30–60 дней релиза 1)

| Метрика | Порог | Рекомендация |
|---|---|---|
| **Retention D7** | <30% | Запустить **P1-01** (письмо себе) + **P1-04** (тихая геймификация) |
| **Retention D7** | 30–50% | Запустить **P1-01** (письмо себе) |
| **Retention D7** | >50% | Оставить текущие механики, не добавлять геймификацию |
| **Доля "близких" в аудитории** | >15% (по UTM/опросам) | Запустить **P1-02** (карта поддержки близкому) |
| **Доля кризисных запросов** | >10% | Запустить **P1-03** (кризисная карточка) |

### Рекомендуемый порядок запуска

**Волна 1 (через 60 дней):**
1. **P1-01** (Письмо себе) — низкая трудоёмкость, высокая ценность для retention
2. **P1-04** (Тихая геймификация) — если retention низкий

**Волна 2 (через 90–120 дней):**
3. **P1-02** (Карта поддержки близкому) — если аудитория близких растёт
4. **P1-03** (Кризисная карточка) — если кризисных запросов много и юридическая проверка пройдена

## Чеклист готовности активности

### Перед запуском любой P1-активности:

- [ ] Проверены метрики релиза 1 (retention/кризисы/аудитория)
- [ ] Контент/шаблоны написаны и вычитаны (тон, дисклеймеры)
- [ ] Юридическая проверка (если применимо)
- [ ] Хранение данных минимизировано (только агрегаты/ID)
- [ ] События аналитики настроены
- [ ] Кризисный режим предусмотрен (если применимо)
- [ ] UX протестирован (easy to disable, no pressure)
- [ ] Команда обучена (если нужна модерация/поддержка)
