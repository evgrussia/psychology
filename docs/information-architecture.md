# IA (Information Architecture) — структура разделов, навигация и маршруты (Release 1)

**Проект:** «Эмоциональный баланс»  
**Версия:** v0.1 (draft)  
**Дата:** 2026-01-06  
**Основано на:** `docs/PRD.md`, `docs/user-flows-cjm.md`, исследования `docs/research/01–13`

## 0) Зачем этот документ
Этот документ фиксирует **информационную архитектуру (IA)**: структуру разделов/страниц, навигацию, URL‑паттерны и правила перелинковки. Это “контракт” для дизайна и разработки, чтобы:
- все сценарии из `docs/user-flows-cjm.md` ложились на понятную карту экранов;
- пользователь за ≤3 клика находил **первый полезный шаг**, **доверие**, **запись**;
- структура была согласованной для SEO, аналитики и будущего расширения.

## 1) Роли и зоны доступа
- **Гость**: публичные страницы, интерактивы без обязательного ввода контакта.
- **Пользователь**: гость или авторизованный (может иметь сохранения/настройки).
- **Клиент**: авторизованный пользователь с записью/оплатой/материалами/дневниками.
- **Админ (owner)**: психолог — полный доступ в админ‑панели.
- **Админ (assistant)**: расписание/лиды/шаблоны, без доступа к чувствительным заметкам (если появятся).
- **Контент‑редактор**: контент/ресурсы/лендинги/интерактивы (тексты), без клиентских данных.

## 2) Глобальные принципы IA (must-have)
- **Time-to-benefit 1–3 минуты**: на главной, лендингах тем, статьях, интерактивах — всегда есть “первый шаг”.
- **Доверие на каждой ключевой странице**: “как проходит консультация”, границы/конфиденциальность, FAQ, честная позиция.
- **Privacy by design**: интерактивы без обязательного контакта; раздельные согласия в формах; минимум хранения (см. `docs/research/10-*`).
- **Кризисный режим**: при триггерах показываем “экстренную помощь” и не делаем “продающие” CTA главными (см. `docs/user-flows-cjm.md`, `docs/research/01-*`, `04-*`).

## 3) Глобальная навигация (Header / Footer)

### 3.1 Header (верхнее меню)
**Обязательные пункты (Release 1):**
- **С чего начать** (выпадающее): “первый шаг” / мини‑диагностики / навигатор состояния / термометр ресурса / скрипты границ / подготовка к первой встрече / мини‑ритуалы / ресурсы
- **С чем я помогаю** (каталог тем)
- **Услуги**
- **Блог**
- **Запись** (постоянная заметная кнопка)

**Условные пункты (если в релизе 1 включаем как отдельные разделы):**
- **Интерактивы** (каталог) — если их будет много и нужны фильтры
- **Мероприятия**

**Утилити (правый край/иконка):**
- **Личный кабинет** (если авторизован) / **Войти**

### 3.2 Footer (подвал)
- **Навигация** (дублирование ключевых разделов)

  - **`/admin/templates/`** — шаблоны сообщений (owner+assistant)

## 5) Стандарты страниц (page patterns) — что обязано быть

### 5.1 Паттерн “Главная / хаб”
- быстрый выбор состояния/темы
- 1–3 “первых шага” (разные форматы) + показ времени (“~60 сек”, “~3–5 минут”)
- доверие (как проходит / границы / FAQ)
- CTA: “Запись” и “Telegram” (оба объяснённые)

### 5.2 Паттерн “Лендинг темы (problem landing)”
Минимум блоков:
- интро (нормализация) + 3–5 признаков (без клиники)
- “что попробовать сейчас” (упражнение/ресурс/мини‑диагностика)
- “как я работаю с этим” + “что будет на первой встрече”
- доверие: границы/конфиденциальность + FAQ
- CTA: запись / Telegram

### 5.3 Паттерн “Статья / ресурс”
- блок “попробовать сейчас” (обязателен)
- 2 CTA (TG/запись) с объяснением
- блок доверия (мини‑профиль автора или trust strip)
- дисклеймеры по редполитике

### 5.4 Паттерн “Квиз/мини‑диагностика”
- превью: цель, время, “не диагноз”, “не экстренная помощь”
- результат: уровень + шаги “сейчас” и “на неделю” + “когда обратиться”
- кризисный хендлинг (при триггерах)

### 5.6 Паттерн “Навигатор состояния” (ветвящийся сценарий)
- короткое превью (время, “не диагноз”, можно остановиться)
- 6–10 шагов выбора (без обязательного текста)
- результат: профиль/ориентир + шаги “сейчас” + план на неделю
- CTA: Telegram (план) / запись

### 5.7 Паттерн “Термометр ресурса”
- 3 простые шкалы (энергия/напряжение/поддержка) + 1–2 выбора
- результат: `resource_level` + 2 шага “сейчас” + (опционально) план на 3 дня
- CTA: мини‑ритуалы / Telegram / запись

### 5.8 Паттерн “Скрипты границ”
- выбор сценария + стиля + цели
- выдача 3–6 фраз (карточки) + “что делать, если давят”
- CTA: копировать / сохранить / Telegram

### 5.9 Паттерн “Подготовка к первой консультации”
- мастер 5–7 шагов, свободный текст не обязателен
- итог: черновик запроса + чек‑лист “как проходит”
- CTA: запись / экспорт (copy/PDF/Telegram)

### 5.5 Паттерн “Запись”
- понятный прогресс/шаги
- прозрачные правила отмен/переноса рядом с оплатой
- “нет слотов” — не тупик (лист ожидания / TG‑консьерж)

## 6) Таксономия и правила перелинковки

### 6.1 Таксономия (единый словарь)
Базовые измерения (см. `docs/research/01-*`):
- **Тема/состояние**: тревога, выгорание, отношения, границы, самооценка (стартовый набор релиза 1)
- **Формат пользы**: статья / микро‑заметка / упражнение / аудио / чек‑лист / квиз
- **Time-to-benefit**: 1–3 мин / 7–10 мин / 20–30 мин / серия/челлендж
- **Цель**: облегчение сейчас / ясность / решение / навыки границ и т.п.

### 6.2 Перелинковка (must-have)
- **Лендинг темы →** 1 интерактив + 2–4 статьи + 2–4 ресурса + 1–2 услуги
- **Статья →** 1 ресурс (практика) + 1 лендинг темы + 1 услуга/запись + (опционально) related‑статьи
- **Ресурс/интерактив →** соответствующий лендинг темы + CTA TG + CTA запись
- **Страница услуги →** 2–3 релевантных темы + “как проходит” + FAQ

## 7) Системные страницы и состояния
- **`/404/`** — страница не найдена (с “С чего начать” + поиск по блогу/ресурсам)
- **`/500/`** — ошибка (бережный текст + “повторить” + “экстренная помощь” ссылкой)
- **Empty state** (нет слотов / нет материалов / нет записей дневника) — отдельные тексты и следующий шаг (см. `docs/research/11-*`)

## 8) Открытые решения (нужно подтвердить перед дизайном)
- Показываем ли **ближайшие слоты публично** (на главной/услугах) или только внутри `/booking/`.
- Нужна ли отдельная страница **`/interactive/`** как каталог, или достаточно "С чего начать" + встроенные модули на страницах.
- Формат Telegram в релизе 1: **канал/бот/гибрид** (влияет на CTA и подтверждение `tg_subscribe_confirmed`).

## 9) Словарь терминов (`/glossary/`) — детали

### 9.1 Зачем
- **SEO**: термины дают дополнительные точки входа (например, "КПТ", "когнитивно-поведенческая терапия", "схема-терапия")
- **Доверие**: показывает экспертность и прозрачность подхода
- **Образование**: помогает пользователям понимать "что это" перед записью

### 9.2 Структура страницы термина

**URL:** `/glossary/{term-slug}/` (например, `/glossary/kbt/`, `/glossary/trevoga/`)

**Минимальные блоки:**
- **Название термина** (H1)
- **Краткое определение** (1–2 предложения, простым языком)
- **Подробное объяснение** (2–3 абзаца):
  - Что это значит в контексте психологии
  - Как это проявляется
  - Когда это может быть полезно/важно
- **Связанные темы/статьи** (перелинковка)
- **Связанные интерактивы/ресурсы** (если применимо)
- **CTA:** "Обсудить на консультации" (запись)

### 9.3 Стартовый набор терминов (P1, не критично для MVP)

**Подходы/методы (если психолог использует):**
- КПТ (когнитивно-поведенческая терапия)
- ACT (терапия принятия и ответственности)
- Схема-терапия
- Психодинамика (если применимо)
- Mindfulness / Осознанность

**Состояния/симптомы:**
- Тревога
- Выгорание
- Панические атаки
- Прокрастинация
- Эмоциональное истощение

**Понятия/техники:**
- Границы (психологические)
- Триггеры
- Копинг-стратегии
- Самосострадание
- Заземление (grounding)

### 9.4 Правила создания терминов

✅ **Простой язык** (без медицинского жаргона)  
✅ **Короткие статьи** (300–500 слов максимум)  
✅ **Дисклеймеры** где нужно ("не диагноз")  
✅ **Перелинковка** на статьи/ресурсы/интерактивы  
✅ **SEO-оптимизация** (мета-описания, синонимы)  

### 9.5 Навигация словаря

**Главная словаря (`/glossary/`):**
- Алфавитный список или фильтры по категориям:
  - Подходы/методы
  - Состояния
  - Понятия/техники
- Поиск по терминам

## 10) Подборки контента (`/curated/`) — детали

### 10.1 Зачем
- **Навигация по большому каталогу:** когда статей/ресурсов много, подборки помогают найти нужное
- **Сценарное мышление:** "что почитать про отношения", "что послушать перед сном"
- **Удержание:** возвращаемость через "что ещё полезного"

### 10.2 Структура подборки

**URL:** `/curated/{collection-slug}/` (например, `/curated/anxiety-starter-pack/`, `/curated/sleep-hygiene/`)

**Минимальные блоки:**
- **Название подборки** (H1)
- **Для кого/зачем** (1–2 предложения)
- **Список материалов** (5–10 элементов):
  - Статьи
  - Упражнения
  - Аудио-практики
  - Чек-листы
  - Квизы/интерактивы
- **CTA:** "Сохранить подборку в Telegram" или "Записаться для разбора"

### 10.3 Типы подборок

**По теме (problem-based):**
- "Стартовый набор: тревога"
- "Что делать с выгоранием"
- "Укрепить границы: практики"
- "Улучшить сон"
- "Отношения: с чего начать"

**По формату (format-based):**
- "Быстрые упражнения (1–3 минуты)"
- "Аудио-практики для дороги/перед сном"
- "Чек-листы для распечатки"
- "Статьи для понимания (10–15 минут чтения)"

**По цели (goal-based):**
- "Подготовиться к первой встрече"
- "Поддержка между сессиями"
- "Самопомощь: с чего начать"
- "Для близких: как поддержать"

**По событию/контексту:**
- "Перед сложным разговором"
- "После конфликта"
- "Когда накрывает тревога"
- "Когда нет сил"

|---|---|---|---|---|

### 10.5 Правила создания подборок

✅ **Фокус и ясность:** одна подборка = одна цель  
✅ **Оптимальный размер:** 5–10 элементов (не перегружать)  
✅ **Разнообразие форматов:** статьи + упражнения + чек-листы  
✅ **Прогрессия:** от простого к сложному (если применимо)  
✅ **Актуальность:** регулярный аудит (раз в квартал)  

### 10.6 Навигация подборок

**Главная подборок (`/curated/`):**
- Карточки подборок с:
  - Название
  - Для кого/зачем (1 предложение)
  - Сколько элементов
  - Темы/теги
  - Время на прохождение (если применимо)
- Фильтры: по теме, по формату, по цели

## 11) Перелинковка (обновлённая с учётом словаря и подборок)

### 11.1 Обязательные связки

- **Лендинг темы →** 1 интерактив + 2–4 статьи + 2–4 ресурса + 1–2 услуги + **1 подборка** + связанные термины словаря
- **Статья →** 1 ресурс (практика) + 1 лендинг темы + 1 услуга/запись + related‑статьи + **1–2 термина словаря** (если упоминаются)
- **Ресурс/интерактив →** соответствующий лендинг темы + CTA TG + CTA запись + **"ещё по теме" (подборка)**
- **Страница услуги →** 2–3 релевантных темы + "как проходит" + FAQ + **подборка "подготовиться"**
- **Термин словаря →** статьи/ресурсы по теме + лендинг темы + **подборка**
- **Подборка →** все элементы внутри + CTA "сохранить/Telegram/запись"

### 11.2 Внутренние ссылки (минимум)

Каждая страница контента должна иметь **минимум 3–5 внутренних ссылок** на релевантный контент (для SEO и навигации).
