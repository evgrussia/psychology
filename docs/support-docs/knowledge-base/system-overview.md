# Обзор системы (System Overview)

Документ описывает высокоуровневую архитектуру и ключевые компоненты проекта «Эмоциональный баланс».

## Технологический стек

*   **Frontend**: Next.js (TypeScript), Tailwind CSS.
*   **Backend**: Django (Python), Django REST Framework.
*   **База данных**: PostgreSQL (основное хранилище).
*   **Кэш и очереди**: Redis.
*   **Инфраструктура**: Nginx, Gunicorn, Docker (в разработке), Ubuntu Server.

## Схема компонентов

1.  **Web-приложение (Frontend)**: Отвечает за интерфейс, прохождение упражнений и флоу записи.
2.  **API (Backend)**: Реализует бизнес-логику, управление пользователями, бронирование и интеграции.
3.  **Интеграции**:
    *   **Google Calendar**: Хранение и синхронизация рабочих часов психологов.
    *   **ЮKassa**: Прием платежей.
    *   **Telegram Bot**: Отправка планов на неделю и напоминаний.
    *   **Email**: Сервисные уведомления.

## Критические пути (Golden Paths)

Для поддержки наиболее важны следующие цепочки действий:
1.  **Auth Flow**: Вход по Magic Link через email.
2.  **Booking Flow**: Выбор слота → Анкета → Оплата → Подтверждение в календаре.
3.  **Interactive Flow**: Прохождение квиза → Получение результата → Сохранение в Telegram.

## Мониторинг

Мы отслеживаем состояние системы через:
*   **Prometheus**: Сбор метрик (CPU, RAM, API Response Time, Error Rate).
*   **Grafana**: Визуализация метрик на дашбордах.
*   **Alertmanager**: Отправка уведомлений о сбоях в Slack.

Полная схема архитектуры доступна в документе `docs/Архитектурный-обзор.md`.
