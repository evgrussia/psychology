# Протокол эскалации (Escalation)

Когда стандартных инструкций (runbooks) недостаточно, проблема передаётся на следующий уровень.

## Уровни инцидентов

### SEV-1: Критический (Critical)
- **Что происходит:** Сайт недоступен, не работает оплата, утечка данных.
- **Кому передать:** DevOps / Lead Developer немедленно.
- **Канал связи:** PagerDuty + Общий звонок.

### SEV-2: Высокий (High)
- **Что происходит:** Не работает важная фича (квизы, Telegram-уведомления), проблемы у группы пользователей.
- **Кому передать:** Backend разработчику.
- **Время реакции:** в течение 30 минут.

### SEV-3: Средний (Medium)
- **Что происходит:** Ошибки в верстке, опечатки, баги у одиночных пользователей, не влияющие на основной флоу.
- **Кому передать:** Разработчику в рамках текущей очереди задач.

## Формат передачи задачи (Handoff)
При эскалации обязательно укажите:
1. `User ID` или `Email` пострадавшего пользователя.
2. `Request ID` из логов (если есть).
3. Описание шагов для воспроизведения проблемы.
4. Ссылка на лог ошибки в мониторинге (Sentry/Grafana).
