# Runbook: Проблемы с записью (Booking Issues)

Этот документ описывает типовые проблемы при записи на консультацию и способы их решения.

## 1. Слот занят (Double Booking)
**Симптомы:** Пользователь видит ошибку «Этот слот уже заняли», хотя в календаре он отображается как свободный.
**Действия:**
1. Проверить Google Calendar специалиста напрямую. Возможно, слот занят личным событием.
2. Проверить таблицу `AvailabilitySlot` в базе через админку. Статус должен быть `AVAILABLE`.
3. Если слот занят в Google, но свободен на сайте — принудительно запустить синхронизацию (через админ-панель).
4. Если пользователь уже начал оплату, слот бронируется на 15 минут (PENDING). Нужно подождать или сбросить бронь, если оплата не прошла.

## 2. Не приходит подтверждение записи
**Симптомы:** Оплата прошла, но в Личном кабинете записи нет и письмо не пришло.
**Действия:**
1. Проверить статус транзакции в ЮKassa.
2. Найти запись в таблице `Appointment`. Если её нет, проверить логи на наличие ошибок при создании `Appointment` после вебхука оплаты.
3. Проверить статус отправки Email в логах почтового сервиса.

## 4. Ошибки синхронизации времени
**Симптомы:** Пользователь жалуется, что время в календаре не совпадает с его часовым поясом.
**Действия:**
1. Проверить настройки часового пояса пользователя в его профиле.
2. Убедиться, что системный часовой пояс (UTC) корректно конвертируется во фронтенде.
3. Попросить пользователя прислать скриншот настроек времени в его системе/браузере.
