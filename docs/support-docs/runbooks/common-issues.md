# Решение типичных проблем (Common Issues Runbook)

Этот документ содержит инструкции по решению наиболее частых проблем, с которыми сталкиваются пользователи.

## 1. Проблемы с аутентификацией

### Симптом: Пользователь не получает код на email
1.  **Проверка логов**: Выполните поиск по email пользователя в системе логирования (см. раздел [Работа с логами](../troubleshooting/logs.md)).
2.  **Статус отправки**: Проверьте статус в панели управления почтовым провайдером (например, SendGrid/Postmark).
3.  **Действие**: Если код отправлен успешно, попросите пользователя проверить папку «Спам». Если ошибка на нашей стороне — создайте тикет для инженеров (L2).

### Симптом: Ошибка «Неверный код»
1.  Убедитесь, что пользователь вводит код из последнего письма (коды имеют срок жизни 10-15 минут).
2.  Проверьте, не заблокирован ли пользователь за подозрительную активность.

## 2. Проблемы с записью и оплатой

### Симптом: Оплата прошла, но запись не подтверждена
1.  **Проверка ЮKassa**: Проверьте статус платежа в личном кабинете ЮKassa по ID транзакции или email.
2.  **Проверка Webhooks**: Убедитесь, что сервер получил уведомление об оплате (логи API).
3.  **Действие**: Если оплата успешна, но статус в БД не обновился — обновите статус вручную через админ-панель (только для L2) и подтвердите запись пользователю.

### Симптом: Нет свободных слотов, хотя календарь психолога пуст
1.  **Синхронизация**: Проверьте статус интеграции с Google Calendar в админ-панели.
2.  **Настройки**: Убедитесь, что слоты не попадают под «буферное время» (минимум 1 час между встречами).

## 3. Проблемы в Личном кабинете

### Симптом: Не отображаются записи в дневнике
1.  Убедитесь, что пользователь авторизован именно под тем email, под которым создавались записи.
2.  Проверьте статус синхронизации фронтенда с API.

---
**Важно**: При общении с пользователем сохраняйте спокойный и бережный тон, даже если ситуация конфликтная. Используйте формулировки из [Контент-гайда](../../Content-Guide-UX-Copywriting.md).
