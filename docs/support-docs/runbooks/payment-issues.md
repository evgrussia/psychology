# Runbook: Проблемы с оплатой (Payment Issues)

Процесс оплаты критически важен. Все проблемы должны решаться с приоритетом HIGH.

## 1. Статус платежа «В обработке» слишком долго
**Симптомы:** Деньги списаны, но статус в системе не меняется более 10 минут.
**Действия:**
1. Проверить личный кабинет ЮKassa на наличие успешной транзакции.
2. Если в ЮKassa платёж успешен, проверить логи вебхуков (`/api/v1/payments/webhook/`).
3. Если вебхук пришёл с ошибкой 500 — исправить причину и повторно отправить вебхук из ЮKassa (или вручную обновить статус в админке).
4. Если вебхука не было — проверить доступность API для внешних запросов.

## 2. Ошибка при переходе к оплате
**Симптомы:** После нажатия «Оплатить» пользователь видит системную ошибку.
**Действия:**
1. Проверить логи Backend на наличие ошибок `YookassaAPIError`.
2. Убедиться, что `ShopID` и `SecretKey` в настройках production корректны.
3. Проверить, что сумма и валюта передаются в правильном формате.

## 3. Возврат средств (Refunds)
**Процедура:**
1. Возвраты осуществляются только через администратора.
2. В админ-панели найти нужный `Payment`.
3. Инициировать возврат через кнопку «Refund» (если интегрировано) или вручную в кабинете ЮKassa.
4. После подтверждения возврата изменить статус `Appointment` на `CANCELLED`.
