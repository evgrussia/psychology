# Отладка проблем (Debugging)

Инструкция по первичному исследованию технических сбоев.

## Работа с логами
Все логи пишутся в формате JSON и доступны в системе сбора логов (Loki/ELK).

### Как искать:
1. **По `request_id`:** Самый эффективный способ проследить путь конкретного запроса через всю систему.
2. **По `user_id`:** Чтобы увидеть все действия пользователя за период.
3. **По уровню `ERROR`:** Чтобы найти необработанные исключения.

## Команды для быстрой проверки (через SSH):
```bash
# Просмотр последних ошибок в логе Django
tail -n 100 /var/log/django/app.log | grep "ERROR"

# Проверка статуса фоновых задач
celery -A config inspect active

# Проверка связности с базой данных
python manage.py check
```

## Анализ трейсов (Tracing)
Используйте Jaeger для визуализации медленных запросов. Это поможет понять, на каком этапе происходит задержка: в самой Django, в запросе к БД или при ожидании ответа от внешней API (например, Google Calendar).
