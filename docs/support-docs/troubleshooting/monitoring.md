# Использование мониторинга (Monitoring)

Мониторинг позволяет обнаружить проблемы раньше, чем о них напишут пользователи.

## Основные инструменты

### Grafana Dashboards
- **Golden Paths:** Показывает воронку от входа до успешной записи. Просадка на любом этапе — сигнал к проверке.
- **API Health:** Процент ошибок 5xx и латентность (p95).
- **Integrations:** Статус связи с Google Calendar, ЮKassa и Telegram.

### Prometheus (Метрики)
Используйте для проверки аномалий:
- Резкий рост потребления памяти/CPU.
- Увеличение количества активных соединений с БД.
- Рост очереди задач в Celery.

### Alertmanager
Все критичные алерты дублируются в Telegram-канал поддержки. 
- Если пришёл алерт `APIHighErrorRate` — это приоритет SEV-1.
- Если `DatabaseSizeHigh` — предупредите DevOps о необходимости расширения диска.
