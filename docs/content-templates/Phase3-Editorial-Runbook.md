# Ранбук Фазы 3: поддержка и рост контента

**Версия:** 1.0  
**Дата:** 2026-01-28  
**Назначение:** операционный ритм и правила наполнения после Фазы 1–2. См. `docs/Content-Filling-Plan.md`, раздел «Фаза 3».

---

## 1. Недельный ритм (целевые объёмы)

| Тип контента | Частота | Где создавать |
|--------------|---------|----------------|
| **Большая статья** (pillar / supporting, 12–18 мин) | 1 в неделю | Django Admin → ContentItemModel, content_type=article |
| **Микро-заметки** (1–2 мин, тег `micro`) | 2–3 в неделю | То же, тег `micro` в tags |
| **Ресурс** (exercise / audio / tool / video) | 1 в неделю | ContentItemModel, content_type: exercise \| audio \| tool \| video |
| **Интерактив** (квиз / навигатор / ритуал) | 1 раз в 1–2 недели | Django Admin → InteractiveDefinitionModel; контент вопросов — фронт или definition_json |

**Практика:** в начале недели зафиксировать план (1 большая, 2–3 микро, 1 ресурс, при необходимости — интерактив). Команда `python manage.py phase3_weekly_check` выводит чеклист и опционально проверяет последние публикации.

---

## 2. Перелинковка (каждая статья и ключевые ресурсы)

- **В теле контента:** 2–5 внутренних ссылок на другие материалы (статьи, ресурсы, темы).
- **Формат ссылок:** Markdown `[Текст](/blog/slug)` или `[Текст](/resources/slug)`; для тем — `/topics/anxiety` и т.д.
- **Рекомендация:** в конце статьи блок «См. также» с 2–5 ссылками по теме (см. seed Фазы 1–2 как образец).

**Проверка:** перед публикацией убедиться, что в `content_body` есть минимум 2 ссылки на внутренние страницы (`/blog/`, `/resources/`, `/topics/`).

---

## 3. CTA (призывы к действию)

В каждой публикуемой статье и ключевом ресурсе:

- **Запись:** ссылка/кнопка на запись (например «Записаться на встречу») с коротким объяснением «зачем» (см. `docs/content-templates/Phase0-Text-Templates.md`, раздел 3).
- **Telegram (по возможности):** опция «Сохранить в Telegram» / «Получить напоминание» с понятной подписью.

Шаблоны формулировок — в `Phase0-Text-Templates.md`, раздел 3.

---

## 4. Обновление лендингов тем

- При появлении новых запросов/трендов (поиск, обратная связь, аналитика) — обновлять лендинги тем (`/topics/[slug]`).
- Проверять раз в месяц: актуальность формулировок, ссылки на свежие статьи и ресурсы по теме.
- Темы (коды): anxiety, burnout, relationships, self-esteem, boundaries, stress.

---

## 5. Чеклист перед публикацией (напоминание)

Перед сохранением и публикацией любого контента:

- [ ] Дисклеймер (по шаблонам Phase0, разд. 1).
- [ ] Тон не директивный; нет запрещённых формулировок.
- [ ] Есть «первый шаг» (упражнение/ресурс/микро-действие).
- [ ] CTA: запись и/или Telegram с понятным «зачем».
- [ ] Перелинковка: 2–5 внутренних ссылок.
- [ ] Для интерактивов/чувствительных тем — проверен кризисный блок и экстренные контакты.

Подробно: `docs/Content-Filling-Plan.md`, раздел 5.
