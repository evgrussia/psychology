# ADR-005: Payment Provider — ЮKassa

## Status
Accepted

## Date
2026-01-26

## Context
Требуется выбор провайдера платежей для приёма оплат за консультации. Требования:
- Работа в РФ
- Поддержка депозитов и полной оплаты
- Webhooks для статусов платежей
- Надёжность и безопасность

## Decision
Использовать **ЮKassa** как провайдер платежей.

## Consequences

### Positive
- Популярный и надёжный провайдер в РФ
- Webhooks для получения статусов платежей
- Поддержка депозитов и полной оплаты
- Хорошая документация и SDK
- Соответствие требованиям безопасности

### Negative
- Привязка к российскому рынку (но это соответствует целям проекта)

### Neutral
- Стандартный выбор для российских проектов

## Alternatives Considered

### Option 1: Stripe
- **Description:** Международный провайдер платежей
- **Pros:** Широкое использование, отличная документация
- **Cons:** Ограниченная поддержка в РФ, не подходит для российского рынка
- **Why not:** Проект ориентирован на российский рынок

### Option 2: Собственная реализация
- **Description:** Прямая интеграция с банками
- **Pros:** Полный контроль
- **Cons:** Сложность реализации, соответствие требованиям безопасности, лицензирование
- **Why not:** Использование готового провайдера безопаснее и быстрее
